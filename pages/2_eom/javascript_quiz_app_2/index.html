Алгоритм RSA – это первый ассиметричный криптографический алгоритм шифрования и подписи с открытым ключом, разработанный Роном Ривестом, Ади Шамиром и Леонардом Адлеманом. Их алгоритм впервые был опубликован в 1977 году. RSA – один из самых популярных и широко используемых алгоритмов шифрования в современной криптографии для цифровой подписи. В основе алгоритма RSA лежит теория чисел.

Дайджест в криптографии результат преобразования входного сообщения произвольной длины в выходную битовую строку фиксированной длины.

RSA также может шифровать и расшифровывать общую информацию для безопасного обмена данными, а также выполнять проверку цифровой подписи.
В алгоритме RSA используются комбинаторные методы генерации ключей и шифрования данных. В отличие от алгоритмов симметричного шифрования, которые имеют только один ключ для шифрования и расшифровки информации, в алгоритме для шифрования данных используется открытый ключ получателя, а для расшифровки данных используется закрытый ключ получателя.

Функция шифрования/дешифрования

После генерации закрытого и открытого ключа их применяют для шифрования, расшифрования, прописывания и проверки подписи. С помощью секретного ключа расшифровывают и подписывают, с помощью открытого зашифровывают и проверяют подпись.
Если открытый текст равен m, зашифрованный текст = me mod n. Если зашифрованный текст – c, открытый текст = cd mod n.
Чтобы лучше понять описанные выше шаги создадим ключи, на примере малых простых чисел, где p = 17 и q = 13. Значение e может быть равно 5, поскольку оно удовлетворяет условию 1 < e < (p-1)(q-1).
Н = р * q = 221
D = е-1mod(p-1)(q-1) = 29
Пара открытых ключей = (221,5)
Пара секретных ключей = (221,29)
Если значение открытого текста(m) равно 10, вы можете зашифровать его, используя формулу me mod n = 82.
Чтобы расшифровать этот зашифрованный текст (c) обратно в исходные данные, вы должны использовать формулу cd mod n = 29.

В целом алгоритм RSA является одним из самых эффективных и надежных алгоритмов шифрования, который активно используется в сфере информационной безопасности и защиты данных. Его использование позволяет обеспечить безопасность и целостность информации от распространенных угроз и атак.  Однако он также имеет некоторые недостатки, в том числе относительно низкую скорость работы и высокие требования к вычислительным ресурсам.
